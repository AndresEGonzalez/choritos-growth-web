Pre-procesamiento

Objetivo

Las mediciones de crecimiento de Perumytilus purpuratus fueron registradas en archivos Excel independientes para cada periodo experimental.
Antes de realizar los análisis, los datos fueron sometidos a un proceso de pre-procesamiento y control de calidad con el objetivo de:

  • Estandarizar la estructura de los datos
  • Generar identificadores únicos por individuo
  • Calcular tasas de crecimiento individuales
  • Detectar posibles errores de registro
  • Identificar y eliminar valores atípicos

Todo el proceso se implementó en R mediante un flujo de procesamiento reproducible.

Estructura de los datos de entrada

Cada archivo Excel contiene una hoja llamada datos, donde cada fila corresponde a una medición individual.

Los metadatos para todos set de datos son:

mo Mes de la medición 1 al 12
yr Año de la medición 2024 al 2026
site Nombre completo del sitio Quintay etc

Flujo de procesamiento

El flujo de procesamiento de datos se resume en la Figura 1.

Creación de identificador único

Cada individuo fue identificado mediante un código único (chorito_id) construido a partir de las variables:

  • sitio
  • color de etiqueta
  • número de etiqueta
  • clase de tamaño
  • tratamiento
  • temporalidad
  • plot
  • reemplazo de pérdida

Esto permite seguir cada individuo a lo largo del experimento.

Cálculo de tasas de crecimiento

Para cada individuo se calcularon:

  • crecimiento absoluto en longitud (mm)
  • crecimiento absoluto en ancho (mm)
  • crecimiento absoluto en peso (gr)

El crecimiento individual se estimó como:

\[ Growth = L_{final} - L_{inicial} \]

La tasa de crecimiento se calculó como:

\[ Rate = \frac{Growth}{Días\ de\ exposición} \]

expresado en mm/día.

Control de calidad de datos

El pipeline incluye tres controles principales.

  1. Detección de duplicados

Se detectan duplicados en dos niveles:

mediciones repetidas para un mismo individuo y fecha

individuos registrados más de una vez en un mismo periodo

  1. Crecimiento negativo

Registros con crecimiento negativo fueron eliminados, ya que generalmente corresponden a:

  • errores de medición

  • errores de digitación

  • daño en la concha

Estos registros se almacenan en un archivo de log para su revisión.

  1. Detección de outliers

Los valores atípicos se identificaron utilizando el criterio de 1.5 × IQR (rango intercuartílico).

Para cada variable:

  • tasa de crecimiento en longitud

  • tasa de crecimiento en ancho

  • tasa de crecimiento en peso

se calcularon los límites: \[ Lower = Q1−1.5×IQR \] \[ Upper=Q3+1.5×IQR \]

Los registros fuera de estos límites fueron removidos del dataset final.

Orden geográfico de los sitios

Para facilitar la generación de figuras, la variable site se transformó en un factor ordenado según su posición geográfica:

  • Pichilemu
  • Matanzas
  • Las Cruces
  • El Quisco
  • Quintay
  • Curaumilla
  • Montemar
  • Los Molles
  • Punta de Talca
  • Guanaqueros
  • Temblador

Esto permite que los gráficos generados en R mantengan automáticamente el orden espacial de los sitios.

Datos generados por el pipeline

El script genera tres datasets principales:

Archivo Contenido
Mediciones_Crecimiento_Ppurpuratus_RAW.rds Mediciones originales consolidadas
Tasas_Crecimiento_Ppurpuratus_RAW.rds Tasas calculadas sin filtrado
Tasas_Crecimiento_Ppurpuratus_CLEAN.rds Dataset final limpio

Además, se genera un archivo Excel con información de control de calidad que contiene:

  • registros con crecimiento negativo

  • valores atípicos eliminados

  • diagnóstico por sitio y temporada
